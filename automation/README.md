# Symphony Automation

This folder is the project management and software delivery playbook for Symphony. It covers the path from idea intake to production review without requiring a specific Jira project, GitHub organization, CI runner, cloud provider, or paid tool.

## Start Here

Use the docs in this order:

1. [SDLC](./sdlc.md) - end-to-end delivery workflow.
2. [Agile](./agile.md) - sprint rhythm, estimation, and definitions.
3. [Jira](./jira.md) - generic issue setup and workflow.
4. [GitHub](./github.md) - branch, issue, PR, review, and release workflow.
5. [CI/CD](./ci-cd.md) - recommended validation and release gates.
6. [Commit Conventions](./commit-conventions.md) - commit message rules.
7. [AI Tools](./ai-tools.md) - current AI coding tool comparison and recommendations.
8. [GitHub Resources](./github-resources.md) - official learning links.

## Operating Rule

Documentation in this folder describes the process. It does not replace human review, credential management, CI configuration, release approval, or production monitoring.
